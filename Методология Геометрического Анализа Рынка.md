## 📋 Аннотация

Данный документ представляет собой комплексную методологию анализа финансовых рынков, основанную на геометрическом подходе к классификации рыночных состояний. Методология прошла эволюцию от концептуальных идей через математическую формализацию к инженерному стандарту с операционной завершённостью.

**Ключевой принцип:**

> ┌─────────────────────────────────────────────────────────────────┐
> │  Рынок нельзя предсказать точно, но можно классифицировать    │
> │  его состояние с высокой надёжностью.                         │
> │                                                                │
> │  Выживание > Прибыль. Адаптация > Прогноз. Структура > Шум.  │
> └─────────────────────────────────────────────────────────────────┘

**Целевое назначение:**
- Построение устойчивых торговых систем
- Управление рисками через мета-индикаторы
- Детекция смены рыночных режимов
- Валидация стратегий на устойчивость к переобучению

## 🔬 Часть 1. Природа Рыночного Шума

> **🔗 Связанные разделы:** [Часть 2](#-часть-2-геометрический-анализ) (Фазовое пространство), [Часть 4](#-часть-4-валидация-и-устойчивость) (Валидация)

### 1.1. Стилизованные Факты Финансовых Рядов

Финансовые временные ряды демонстрируют следующие подтверждённые свойства:

| Свойство | Описание | Практическое Следствие |
| :--- | :--- | :--- |
| **Толстые хвоста (Fat Tails)** | Распределение доходностей имеет эксцесс в 3-10 раз выше нормального | События «10 сигма» происходят раз в 5-10 лет, а не раз в 10 000 лет |
| **Кластеризация волатильности** | Периоды затишья и шторма группируются (ARCH/GARCH эффекты) | Волатильность предсказуема лучше, чем направление цены |
| **Долгосрочная память** | Показатель Хёрста (H) отклоняется от 0.5 (случайность) | Рынок имеет инерцию (H > 0.5) или возврат к среднему (H < 0.5) |
| **Мультифрактальность** | Свойства масштабируются нелинейно across таймфреймов | Стратегия должна работать на нескольких масштабах одновременно |
| **Скачкообразность (Jumps)** | Цены движутся дискретно, с разрывами в моментах шоков | Модели должны учитывать разрывы, а не только непрерывные движения |

### 1.2. Типология Шума

| Тип | Присутствие в Финансах | Применение |
| :--- | :--- | :--- |
| **Белый шум** | ❌ Почти никогда (только аппроксимация на микро-интервалах) | База для гипотезы эффективного рынка (EMH) |
| **Розовый шум (1/f)** | ✅ **Часто**. Волатильность и объемы | Объясняет кластеризацию волатильности |
| **Коричневый шум** | ⚠️ **Частично**. Цены часто моделируют как GBM | Основа Black-Scholes, но не учитывает «толстые хвосты» |
| **Чёрный шум** | ✅ **Периодически**. Крахи, флэш-крэши | Требует моделей с прыжками (Jump-diffusion) |

> **Вывод 1:** Любая модель, предполагающая нормальное распределение, **систематически недооценивает риски**.

> **🔗 Связанные разделы:** [Часть 3](#-часть-3-классификация-режимов) (Классификация), [Часть 5](#-часть-5-нормализация-данных) (Нормализация), [Часть 6](#-часть-6-калибровка-порогов) (Калибровка)

### 2.1. Метод Вложения Такенса

Преобразование одномерного временного ряда в многомерную траекторию:

$$ \Large \boxed{P_t = (x_t, x_{t-\tau}, x_{t-2\tau}, x_{t-3\tau}, ...)} $$
> **[Формула 1](#глоссарий-обозначений):** Метод вложения Такенса

**Критические параметры:**

| Параметр | Обозначение | Рекомендация | Метод Выбора |
| :--- | :--- | :--- | :--- |
| **Задержка** | τ (tau) | Оптимальная, не последовательная | Минимум mutual information (Fraser & Swinney, 1986) |
| **Размерность вложения** | m (embedding dim) | 3-5 измерений (не больше!) | Тест на насыщение фрактальной размерности |
| **Нормализация** | — | Ранги или лог-доходности | Скользящее окно для устранения look-ahead bias |

**Практические значения:**
- Для дневных данных: τ ≈ 3-5 дней
- Для внутридневных: τ ≈ 10-20 баров
- Размерность m > 7 ведёт к «проклятию размерности»

### 2.1.1. Стабилизация τ в Продакшене

> ┌─────────────────────────────────────────────────────────────────┐
> │  ⚠️ Критическое дополнение для Production-систем              │
> │                                                                 │
> │  Параметр τ, рассчитанный через mutual information, может     │
> │  «прыгать» на скользящем окне из-за нестационарности ряда.   │
> │  Частый пересчёт τ создаёт нестабильность фазового            │
> │  пространства.                                                 │
> └─────────────────────────────────────────────────────────────────┘

**Рекомендация по стабилизации:**

| Режим | Частота пересчёта τ | Применение |
| :--- | :--- | :--- |
| **Консервативный** | 1 раз в 252 торговых дня | Для долгосрочных стратегий (swing, position) |
| **Адаптивный** | 1 раз в 63 торговых дня (квартал) | ✅ **Рекомендуется** для большинства систем |
| **Агрессивный** | 1 раз в 21 торговый день (месяц) | Для внутридневных систем с высокой адаптивностью |

**Процедура:**
```
1. Рассчитать τ на историческом окне (минимум 252 дня)
2. Зафиксировать значение на период N дней
3. Между пересчётами использовать фиксированное τ
4. Мониторить стабильность τ: если коэффициент вариации 
   (std/mean) при пересчётах > 0.3 → пересмотреть метод
```

> **Это предотвращает «дергание» фазового пространства и обеспечивает стабильность метрик.**

### 2.2. Метрики в Пространстве Состояний

| Метрика | Формула | Интерпретация | Сигнал |
| :--- | :--- | :--- | :--- |
| **Евклидово Расстояние** | $D = \sqrt{\sum(x_i - y_i)^2}$ | Скорость изменения состояния | Резкий рост D = смена режима |
| **Косинусное Расстояние** | $\cos\theta = \frac{\vec{V_t} \cdot \vec{V_{t-1}}}{|\vec{V_t}| \cdot |\vec{V_{t-1}}|}$ | Сохранение инерции | θ ≈ 0° = тренд, θ ≈ 180° = разворот |
| **Расстояние Махаланобиса** | $D_M = \sqrt{(x-y)^T \Sigma^{-1} (x-y)}$ | Учёт корреляций между измерениями | ✅ **Рекомендуется для m ≥ 4** |
| **Расстояние до Центра** | $D_c = |P_t - \text{Centroid}_N|$ | Отклонение от нормы | Работает как осциллятор |
| **Энергия (Power)** | $E = \frac{1}{N}\sum x_i^2$ | Сила движения | E ≈ 1 = тренд, E ≈ 0 = флэт |

### 2.2.1. Выбор Метрики в Зависимости от Размерности

| Размерность (m) | Рекомендуемая Метрика | Альтернатива | Комментарий |
| :--- | :--- | :--- | :--- |
| **m = 2-3** | Евклидово расстояние | — | Простота, низкие вычислительные затраты |
| **m = 4-5** | Косинусное расстояние | Евклид (с осторожностью) | Борьба с проклятием размерности |
| **m ≥ 6** | Махаланобиса / TDA-метрики | Косинус | Учёт корреляций между измерениями, требуется больше данных |

> **Важно:** Евклидово расстояние в высоких размерностях (m > 5) становится малоинформативным — все точки оказываются примерно на одинаковом расстоянии друг от друга.

### 2.2.2. Сводная Таблица Выбора Метрик

| Размерность (m) | Рекомендуемая Метрика | Альтернатива | Комментарий |
| :--- | :--- | :--- | :--- |
| **2-3** | Евклидово расстояние | — | Просто, быстро, информативно |
| **4-5** | Косинусное расстояние | Евклид (с осторожностью) | Борьба с проклятием размерности |
| **≥6** | Махаланобиса / TDA | Косинус | Требует больше данных, учёт корреляций |

> **Примечание:** При m ≥ 6 все точки в фазовом пространстве стремятся к одинаковому расстоянию друг от друга. Используйте TDA (персистентную гомологию) для детекции топологических особенностей вместо метрик расстояния. **TDA вычислительно сложна — для реального времени могут потребоваться приближённые методы.**

### 2.3. Обработка Гэпов

> **⚠️ Важное примечание для рынков с гэпами (фондовые индексы, отдельные акции):**
>
> Ценовые разрывы между сессиями создают ложные скачки в Евклидовом расстоянии. Система может интерпретировать их как резкую смену состояния.

**Рекомендация по обработке:**

| Сценарий | Решение |
| :--- | :--- |
| **Стратегии без overnight** | Используйте цены открытия как непрерывный ряд |
| **Интрадей-стратегии** | Детектируйте гэпы: `|open_t - close_{t-1}| / close_{t-1} > threshold` |
| **Обработка гэпов** | Исключайте такие точки из анализа переходов ИЛИ обрабатывайте как отдельный класс событий «Jump» |
| **Альтернатива** | Используйте логарифмические доходности: `log(close_t) - log(close_{t-1})` |

### 2.4. Типы Входных Данных

> **⚠️ Выбор частоты и типа цен критически влияет на параметры τ и m.**

| Тип Данных | Рекомендуемые Параметры | Применение |
| :--- | :--- | :--- |
| **Дневные (Close)** | τ = 3-5, m = 3-4 | Swing-трейдинг, позиционные стратегии |
| **Дневные (OHLC)** | τ = 2-4, m = 4-5 | Учёт внутридневной волатильности |
| **Внутридневные (1-min)** | τ = 10-20, m = 3-4 | Интрадей-трейдинг, скальпинг |
| **Внутридневные (Tick)** | τ = 50-100, m = 3-5 | HFT-системы, маркет-мейкинг |
| **VWAP-бары** | τ = 5-10, m = 3-4 | ✅ **Рекомендуется** для минимизации шума |

**Важное замечание:**
- **Close-цены** — наиболее стабильны, но теряют внутрибарную информацию
- **OHLC-векторы** — больше информации, но выше шум и требования к вычислительным ресурсам
- **VWAP** — лучший компромисс для институциональных стратегий

### 2.4.1. Специфика для HFT-Систем

| Параметр | Рекомендация для HFT | Обоснование |
| :--- | :--- | :--- |
| **Тип данных** | Tick / 1-сек бары | Требуется максимальная гранулярность |
| **τ** | 50-100 тиков | Больше данных для стабильности mutual information |
| **m** | 3-4 | Выше размерность → больше вычислительная сложность |
| **Метрика** | Косинусное расстояние | Устойчивость к масштабу на микро-интервалах |
| **Валидация** | Tick Shuffle (обязательно) | Детекция артефактов микроструктуры |
| **Ликвидность** | Стакан (Level 2/3) | Контроль наполнения на противоположной стороне |
| **Исполнение** | Maker-ордера, rebate | Минимизация транзакционных издержек |

### 2.5. Подтверждённые Результаты Исследований

| Исследование | Метод | Результат |
| :--- | :--- | :--- |
| **Recurrent Quantification Analysis (RQA)** | Матрица расстояний между точками траектории | Выявляет ламинарные фазы (тренды) лучше скользящих средних |

## 🎯 Часть 3. Классификация Режимов (Pattern Matching)

> **🔗 Связанные разделы:** [Часть 2](#-часть-2-геометрический-анализ) (Фазовое пространство), [Часть 7](#-часть-7-мониторинг-стабильности) (Мониторинг)

### 3.1. Метод Скалярного Произведения (Dot Product)

Сравнение текущего вектора состояния $\hat{X}$ с эталонными шаблонами $\hat{W}$:

$$ \Large \boxed{\text{Signature} = \hat{X} \cdot \hat{W} = \cos(\theta) \in [-1:1]} $$
> **[Формула 2](#глоссарий-обозначений):** Скалярное произведение для классификации режимов

**Критическое требование:** Векторы должны быть нормализованы к единичной длине перед умножением.

### 3.2. Эталонные Шаблоны

| Шаблон | Вектор | Детектирует | Режим Стратегии |
| :--- | :--- | :--- | :--- |
| **Синхронный Рост** | `[1, 1, 1, ..., 1]` | Все активы сильно растут | ✅ Трендовая Long |
| **Синхронный Обвал** | `[-1, -1, ..., -1]` | Все активы сильно падают | ✅ Трендовая Short / Hedge |
| **Дивергенция** | `[1, -1, 1, -1...]` | Альтернирующее движение (рынок «пилит») | ✅ Осциллятор / Mean Reversion |
| **Фокус на Группе** | `[0, 0, 1, 1, 0...]` | Только выбранные активы активны | ✅ Секторальная ротация |

### 3.2.1. Логика Выбора Шаблонов

**Вопрос:** Являются ли шаблоны жёстко заданными или система выбирает наиболее подходящий?

**Ответ:** Гибридный подход.

| Уровень | Описание | Применение |
| :--- | :--- | :--- |
| **Базовые шаблоны** | Жёстко заданные векторы (Раздел 3.2) | Первичная классификация режима |
| **Адаптивные центроиды** | Центроиды кластеров из исторических данных | Уточнение режима под конкретный актив |
| **Динамические шаблоны** | Скользящее среднее векторов текущего режима | Для долгосрочной адаптации |

**Процедура выбора:**
```
1. Рассчитать Signature для всех базовых шаблонов
2. Выбрать шаблон с максимальным |Signature|
3. Если |Signature| < 0.5 → режим «Неопределённость» (Не торговать)
4. Если |Signature| ≥ 0.5 → активировать соответствующую суб-стратегию
```

> **Мета-сигнал управляет семейством суб-стратегий, а не является прямым сигналом входа.**

### 3.3. Модуляция Энергией

**Критическое уточнение:** Для Dot Product используется **нормализованный вектор**, для Energy — **исходный (ненормализованный) вектор**.

$$ \Large \boxed{\text{PowerSignal} = \text{Signature} \times \text{Energy}_{\text{smoothed}}} $$
> **[Формула 3](#глоссарий-обозначений):** Модуляция сигнала энергией

| Компонент | Данные | Отвечает за | Применение |
| :--- | :--- | :--- | :--- |
| **Signature** | Нормализованный вектор ($|\vec{V}| = 1$) | «Что делать?» (направление, режим) | Выбор стратегии |
| **Energy** | Исходный вектор доходностей, сглаженный EMA | «Как сильно?» (объём, агрессия) | Размер позиции |

**Практическая реализация:**
```python
# Для направления (нормализуем к единичной длине)
direction = dot_product(normalize(current_vector), template)

# Для силы (берем ИСХОДНЫЙ вектор, сглаженный EMA!)
energy_raw = energy_filter(current_vector_raw) 
energy_smoothed = EMA(energy_raw, span=5)  # Сглаживание для стабильности

# Итоговый сигнал для размера позиции
position_size = base_size * direction * energy_smoothed
```

> **Важно:** 
> 1. Если использовать нормализованный вектор для Energy, его энергия всегда будет равна 1.
> 2. Использование сырого Energy_raw приведёт к излишней нервозности позиции. EMA сглаживает «дрожание».

### 3.4. Подтверждённая Эффективность

| Применение | Точность | Ограничения |
| :--- | :--- | :--- |
| **Детекция Аномалий** | 85-95% за 30-120 минут до скачка | Требует калибровки порогов |
| **Классификация Тренд/Флэт** | 75-85% на спокойных рынках | Ломается при структурных сдвигах |
| **Поиск Исторических Аналогов** | 60-70% точность направления | Не предсказывает величину движения |

> **Вывод 3:** Скалярное произведение — мощный инструмент для **классификации режимов**, но не для **предсказания цены**.

## 🛡️ Часть 4. Валидация и Устойчивость Стратегий

> **🔗 Связанные разделы:** [Часть 6](#-часть-6-калибровка-порогов) (Калибровка), [Часть 10](#-часть-10-отсеянные-идеи) (Отсеянные идеи)

### 4.1. Проблема Переобучения (Data Snooping Bias)

Если стратегия подобрана на исторических данных, **подавляющее большинство оптимизированных стратегий не показывают результатов вперёд** и являются статистическими аномалиями.

**Решение: Walk-Forward Analysis**

```
1. Разделить данные на In-Sample (обучение) и Out-of-Sample (тест)
2. Оптимизировать стратегию на In-Sample
3. Протестировать на Out-of-Sample
4. Повторить для 100+ случайных подвыборок
5. Принять стратегию только если p-value < 0.05 на OOS
```

### 4.2. Дерево Устойчивости (Рекурсивное Разбиение)

Разбиение данных на части для проверки стабильности результатов:

| Паттерн Дерева | Интерпретация | Вердикт |
| :--- | :--- | :--- |
| **Все узлы зелёные (+)** | Стратегия устойчива во всех подпериодах | ✅ Возможно, реальное преимущество |
| **Хаос (+/-)** | Результаты случайны, нет стабильности | ❌ Переобучение / Удача |
| **Время X работает, Y нет** | Стратегия зависит от рыночного режима | ⚠️ Нужен фильтр режимов |
| **Глубина > 3 ломается** | Статистическая значимость падает на малых выборках | ⚠️ Недостаточно данных |

### 4.3. Тест на Суррогатных Данных

**Критическое требование:** Генерация случайных данных должна сохранять стилизованные факты рынка.

| Метод | Что Сохраняет | Сложность | Рекомендация | Python Библиотека |
| :--- | :--- | :--- | :--- | :--- |
| **GARCH-симуляция** | Кластеризацию волатильности, толстые хвосты | ⭐⭐ Средняя | ✅ Минимум | `arch` |
| **AAFT** | Спектр мощности + распределение | ⭐⭐⭐ Высокая | ✅ Рекомендуется | `nolds` |
| **IAAFT** | Спектр + распределение + нелинейности | ⭐⭐⭐⭐ Очень высокая | ✅✅ Идеал | `surrogate` |
| **Bootstrapped Blocks** | Автокорреляцию на коротких лагах | ⭐⭐ Средняя | ⚠️ Базовый | `statsmodels` |
| **Tick Shuffle** | Распределение доходностей (для HFT) | ⭐ Низкая | ✅ Для HFT-валидации | `numpy` |

> **⚠️ Важное предостережение по IAAFT:**
>
> Метод IAAFT блестяще сохраняет линейный спектр мощности, но **разрушает нелинейные зависимости**. Если ваша торговая стратегия (особенно на HFT-горизонтах) основана на нелинейных паттернах — например, на предсказании микро-импульсов или специфических формациях в фазовом пространстве — прохождение IAAFT-теста является **необходимым, но недостаточным условием** устойчивости.

**Дополнительная рекомендация для HFT-систем:**

Для проверки устойчивости к микроструктурным шумам рекомендуется использовать **Тест Перемешивания Тиков (Tick Shuffle)**. Этот метод сохраняет эмпирическое распределение доходностей, но разрушает временную упорядоченность на самом малом масштабе.

| Результат Tick Shuffle | Интерпретация |
| :--- | :--- |
| **Стратегия прибыльна** | ❌ Ловит артефакт микроструктуры (особенность записи данных) |
| **Стратегия убыточна** | ✅ Зависит от временной структуры (хороший знак) |

> **Вывод 4:** **Избегание ложных путей** важнее поиска волшебной таблетки. Дерево устойчивости — это машина по отсеву иллюзий.

## 📊 Часть 5. Нормализация Данных

> **🔗 Связанные разделы:** [Часть 2](#-часть-2-геометрический-анализ) (Фазовое пространство), [Часть 3](#-часть-3-классификация-режимов) (Классификация)

### 5.1. Сравнение Методов

| Метод | Когда Работает | Когда НЕ Работает | Рекомендация |
| :--- | :--- | :--- | :--- |
| **MinMax [-1:1]** | Для нейросетей, важен диапазон активаций | При выбросах (крах «схлопывает» шкалу) | ❌ Не рекомендуется для трейдинга |
| **Z-score** | Для статистических моделей, важна дисперсия | Для нестационарных рядов (среднее и std «плывут») | ⚠️ С скользящим окном |
| **Ранговая** | При сильных выбросах и «тяжёлых хвостах» | Теряется информация о величине экстремумов | ✅ **Рекомендуется** |
| **Логарифмы** | Для отношений цен (сохраняет математику) | Требует положительных цен | ✅ **Рекомендуется** для доходностей |

### 5.2. Правило Адаптивной Нормализации

**Критическое требование:** Все статистики для нормализации должны считаться **только на прошлых данных**.

| Метод | Формула | Когда Использовать |
| :--- | :--- | :--- |
| **Expanding Window** | $\mu_t = \frac{1}{t}\sum_{i=1}^{t} x_i$ | Для долгосрочных стратегий |
| **Rolling Window** | $\mu_t = \frac{1}{w}\sum_{i=t-w}^{t} x_i$ | ✅ **Рекомендуется** для адаптивных систем |
| **Exponential Weight** | $\mu_t = \alpha x_t + (1-\alpha)\mu_{t-1}$ | Для быстрой адаптации |

### 5.3. Преимущество Ранговой Нормализации

Ранговая нормализация **автоматически** решает проблему look-ahead, если ранг считается в скользящем окне:

$$ \text{rank}_t = \text{rank}(x_t, \{x_{t-w}, ..., x_t\}) $$

$$ \text{normalized}_t = \frac{2 \cdot \text{rank}_t}{w} - 1 $$

**Двойное преимущество:**
1.  Устойчивость к выбросам (робастность)
2.  Отсутствие заглядывания в будущее (по дизайну)

> **Вывод 5:** Нормализация + Отношения = Несовместимы без дополнительных преобразований. Правильный порядок: Логарифмы → Разности → Нормализация разностей.

## ⚙️ Часть 6. Калибровка Порогов

> **🔗 Связанные разделы:** [Часть 5](#-часть-5-нормализация-данных) (Нормализация), [Часть 7](#-часть-7-мониторинг-стабильности) (Мониторинг)

### 6.1. Принцип Перцентильной Калибровки

Все пороги в системе должны быть **относительными**, а не абсолютными.

| Метрика | Плохой Порог | Хороший Порог |
| :--- | :--- | :--- |
| **Энергия** | `if Energy > 0.5` | `if Energy > percentile(Energy_history, 80)` |
| **Дистанция** | `if Distance > 2.0` | `if Distance > mean(Distance) + 2*std(Distance)` |
| **Signature** | `if Signature > 0.7` | `if Signature > percentile(Signature_history, 95)` |

### 6.2. Процедура Калибровки

```
1. Собрать историю метрик за N периодов (минимум 252 торговых дня)
2. Построить эмпирическое распределение каждой метрики
3. Выбрать пороги как перцентили распределения:
   - Консервативный: 95-й перцентиль (редкие сигналы, высокая точность)
   - Агрессивный: 80-й перцентиль (частые сигналы, больше шума)
4. (Критическое требование) Провести walk-forward валидацию на множестве эпох.
   Разбейте историю на декадные окна (по 2-3 года). Если параметры показывают
   устойчивость в 8 из 10 окон — хорошо. Если оптимальны только для последних
   5 лет (период QE) — сигнал о подгонке под эпоху. Не фиксируйте такие параметры.
5. Зафиксировать ПАРАМЕТРЫ РАСЧЕТА порогов (длина окна, уровень перцентиля)
6. Абсолютное значение порога пересчитывается ежедневно на основе скользящего окна
7. Мониторить дрейф порогов (см. Часть 7)
```

> **Критическое уточнение:** Фиксируются **параметры расчета** (окно 252 дня, перцентиль 95%), а не абсолютные значения порогов. В продакшене порог должен пересчитываться ежедневно (rolling), иначе адаптивность теряется.

> **⚠️ Защита от «выживательного смещения»:**
>
> Параметры (252 дня, 95-й перцентиль) могут быть подобраны под конкретную историческую эпоху (например, 2015-2024: низкие ставки, QE). Walk-forward на множестве эпох защищает от этого.

**Важно:** Пороги не оптимизируются на прибыль! Они оптимизируются на **стабильность статистических свойств**.

## 🔍 Часть 7. Мониторинг Структурной Стабильности (Production)

> **🔗 Связанные разделы:** [Часть 6](#-часть-6-калибровка-порогов) (Калибровка), [Часть 9.4](#-для-риск-менеджера-валидация-сигнала-ликвидностью) (Ликвидность)

### 7.1. Проблема Дрейфа Порогов

Даже перцентильные пороги могут постепенно смещаться при структурных изменениях рынка. Это создаёт риск «тихого вырождения» стратегии.

### 7.2. Формула Детекции Структурного Слома

$$ \Large \boxed{\text{Drift}_t = \frac{\text{MA}_{60}(\text{Threshold}_t) - \text{Mean}_{252}(\text{Threshold})}{\text{Std}_{252}(\text{Threshold})}} $$
> **[Формула 4](#глоссарий-обозначений):** Детекция структурного дрейфа (в единицах σ — стандартных отклонений)

**Расшифровка обозначений:**
- **MA₆₀(Threshold_t)** — скользящее среднее за 60 дней от текущего ежедневно пересчитываемого значения порога
- **Mean₂₅₂(Threshold)** — среднее арифметическое значение порога за последние 252 дня (исторический эталон)
- **Std₂₅₂(Threshold)** — стандартное отклонение порога за последние 252 дня
- **Единицы измерения:** Drift измеряется в стандартных отклонениях (σ)

| Значение Drift | Интерпретация | Действие |
| :--- | :--- | :--- |
| **|Drift| < 1.0σ** | Норма | Продолжать торговлю |
| **1.0σ ≤ |Drift| < 2.0σ** | Предупреждение | Снизить размер позиции на 50% |
| **|Drift| ≥ 2.0σ** | **Структурный слом** | **STOP. Перейти на ручное управление** |

### 7.3. Скорость Изменения Дрейфа (Предвестник Шока)

Абсолютное значение Drift_t показывает, насколько мы отклонились от нормы. Однако для раннего обнаружения быстроразвивающихся кризисов (flash crashes) критически важна **скорость** этого отклонения.

$$ \Large \boxed{\text{DriftSpeed}_t = \frac{|\text{Drift}_t| - |\text{Drift}_{t-w}|}{w}} $$
> **[Формула 5](#глоссарий-обозначений):** Скорость изменения дрейфа (w = 5 дней по умолчанию, параметризуемо)

*(Оценка изменения за w торговых дней, по умолчанию w = 5)*

| Значение DriftSpeed | Интерпретация | Действие |
| :--- | :--- | :--- |
| **Рост на > 0.3σ за 5 дней** | Аномально быстрое изменение структуры | Превентивно снизить размер позиции на 25% даже при Drift < 1.0σ |

> **Этот индикатор позволяет реагировать на сам факт ускорения изменений, не дожидаясь, пока они станут критическими.**

### 7.4. Процедура Экстренной Перекалибровки

```
1. При |Drift| ≥ 2.0σ → Немедленная пауза торговли
2. Анализ причин дрейфа (регуляторные изменения, кризис, etc.)
3. Пересчёт истории метрик (отбросить старые данные)
4. Рекалибровка порогов на новых данных (минимум 60 дней)
5. Тест на суррогатных данных с новыми параметрами
6. Поэтапный возврат к торговле (25% → 50% → 100%)
```

> **Вывод 6:** Версия 1.1 — это спецификация для разработки. Версия 1.9 — это спецификация для публикации и выживания в продакшене.

## 📈 Часть 9. Практические Рекомендации

> **🔗 Связанные разделы:** [Часть 8](#-часть-8-архитектура-системы) (Архитектура), [Часть 10](#-часть-10-отсеянные-идеи) (Отсеянные идеи)

### 9.1. Для Исследователя

1.  **Начните с энергетического фильтра (Σx²)** — это даст 80% пользы при 20% усилий.
2.  **Внедрите тест на случайных данных** — отбрасывайте стратегии, работающие на шуме.
3.  **Используйте ранговую нормализацию** — она устойчивее к выбросам, чем MinMax.
4.  **Стройте фазовое пространство с 3-5 измерениями** — больше не даёт прироста информации.
5.  **Ведите журнал режимов** — отмечайте, в каких условиях стратегия работает/ломается.
6.  **Выбирайте τ через mutual information** — не используйте последовательные лаги.
7.  **Стабилизируйте τ** — пересчитывайте не чаще 1 раза в квартал.
8.  **Для m ≥ 4 используйте косинусное расстояние** — евклидово теряет информативность.

### 9.2. Для Практика

1.  **Не ищите «святой грааль»** — ищите **устойчивость** в широком диапазоне параметров.
2.  **Управляйте риском через мета-индикаторы** — размер позиции = функция от энергии рынка.
3.  **Переключайте стратегии по режимам** — трендовая для тренда, осциллятор для флэта.
4.  **Мониторьте энтропию портфеля** — резкий рост = снижение позиции.
5.  **Валидируйте на подпериодах** — если стратегия ломается на отдельных участках, она не надёжна.
6.  **Следите за дрейфом порогов** — |Drift| > 2.0σ = немедленная пауза.
7.  **Используйте VWAP-бары** — лучший компромисс между шумом и информацией.

### 9.4. Для Риск-Менеджера: Валидация Сигнала Ликвидностью

> **Геометрический анализ дает сигнал о намерении. Рынок дает возможность исполнить его.**
>
> Никакой сигнал не должен быть передан в торговый модуль без прохождения фильтра ликвидности.

**Правило "Исполнимости":**

| Фильтр | Консервативный | Агрессивный | Действие |
| :--- | :--- | :--- | :--- |
| **Объема (ADV)** | Позиция > 5% от ADV | Позиция > 1-2% от ADV | Блокировка ИЛИ разбивка на части (VWAP) |
| **Participation Rate** | < 5% от минутного объема | < 10% от минутного объема | Контроль скорости входа |
| **Спреда** | Спред > 0.1 × ATR | Спред > 0.05 × ATR | Запрет торговли |
| **Стакана (HFT)** | Недостаточный объем на противоположной стороне | — | Блокировка или уменьшение размера заявки |

**Дополнительные требования:**

1.  **Market Impact:** В бэктестах добавляйте 10-30% к комиссиям для моделирования рыночного воздействия.
2.  **Кризисная ликвидность:** В периоды высокой волатильности (Energy > 95-й перцентиль) ужесточайте фильтры в 2 раза.
3.  **Исполнение:** Используйте алгоритмы VWAP, Iceberg, TWAP для минимизации market impact на крупных позициях.

> **Это превращает теоретически прибыльную стратегию в практически реализуемую.**

## ⚠️ Часть 10. Отсеянные Идеи

### 10.1. Идеи, Не Прошедшие Валидацию

| Идея | Причина Отказа | Альтернатива |
| :--- | :--- | :--- |
| **Прямой прогноз цены через Фурье** | Экстраполяция гармоник даёт ошибки 40-80% из-за нестационарности | Использовать спектр для фильтрации шума, не прогноза |
| **Рекурсивная пирамида отношений до 1 числа** | Усиление шума на каждом уровне, чувствительность к выбросам | Остановиться на 2-3 уровне (ускорение, рывок) как фильтре |
| **Полный перебор всех перестановок (N>12)** | Комбинаторный взрыв, не реализуемо | Монте-Карло выборка (10 000 перестановок) |
| **Нормализация MinMax перед отношениями** | Деление на ноль, потеря масштаба, look-ahead bias | Логарифмы + разности или ранговая нормализация |
| **Одна стратегия для всех режимов** | Рынок меняет свойства, универсальной стратегии нет | Банк стратегий + детектор режимов для переключения |
| **Фиксированные абсолютные пороги** | Не адаптируются к смене волатильности | Перцентильная калибровка на истории |
| **Простое перемешивание для валидации** | Уничтожает кластеризацию волатильности | GARCH/IAAFT/Tick-Shuffle суррогатные данные |
| **Частый пересчёт τ (каждый бар)** | Нестабильность фазового пространства, «дергание» метрик | Пересчёт τ 1 раз в квартал с фиксацией значения |

### 10.2. Типичные Ошибки При Реализации

| Ошибка | Симптом | Решение |
| :--- | :--- | :--- |
| **MinMax-нормализация** | Стратегия ломается на крахах | Использовать ранговую нормализацию |
| **Частый пересчёт τ** | Метрики «дергаются», ложные сигналы | Фиксировать τ на 63 дня (квартал) |
| **Energy на нормализованных данных** | Энергия всегда = 1, нет модуляции | Считать Energy на исходных доходностях |
| **Look-ahead в нормализации** | Стратегия работает только на истории | Использовать только скользящие окна |
| **Игнорирование гэпов** | Ложные детекции смены режима | Лог-доходности или явная детекция |
| **Оптимизация порогов на прибыль** | Переобучение на историю | Оптимизировать на стабильность статистик |

## 📎 Приложение A: Пример Расчёта Drift

**Исходные данные (порог Energy за 252 дня):**
- Mean₂₅₂ = 0.65
- Std₂₅₂ = 0.12
- MA₆₀(Threshold_t) = 0.80 (текущее 60-дневное среднее)

**Расчёт Drift:**
$$\text{Drift}_t = \frac{0.80 - 0.65}{0.12} = 1.25σ$$

**Интерпретация:** 1.0σ ≤ |Drift| < 2.0σ → **Предупреждение** (снизить позицию на 50%)

**Расчёт DriftSpeed (за 5 дней):**
- |Drift_t| = 1.25σ
- |Drift_{t-5}| = 0.45σ

$$\text{DriftSpeed}_t = \frac{1.25 - 0.45}{5} = 0.16σ/день$$

**Интерпретация:** Рост на 0.8σ за 5 дней > 0.3σ → **Превентивное снижение на 25%**
